## 1. 프로젝트 비전 및 목표

### 1.1 비전
제품 기획 초기단계에서 실제 제품과 매우 유사한 비주얼 목업을 빠르게 생성하여, 디자이너와 기획자의 커뮤니케이션 효율을 극대화하는 AI 기반 목업 생성 도구.

### 1.2 핵심 목표
1. **캐릭터 IP 변환**: 기존 출시 제품 이미지에서 캐릭터 IP를 참조하여 변경 적용한 제품 예상 목업 생성
2. **2D → 3D 실사화**: 디자이너가 2D로 그린 디자인을 고퀄리티의 실제 제품 같은 3D 목업으로 변환
3. **일관된 품질 유지**: 형태, 질감, IP 일관성을 유지하면서 재현 가능한 결과물 생성

### 1.3 결과물 활용 용도
- 협력사/바이어와의 디자인 컨셉 공유
- 기획/마케팅/영업 간 커뮤니케이션
- 내부 디자인 검토

## 3. 핵심 기능 정의

### 3.1 기능 A: 캐릭터 IP 변경
**설명**: 다른 캐릭터 IP가 적용된 실제 제품에서 캐릭터를 변경

**입력**:
- 원본 제품 이미지 (다른 캐릭터가 적용된 상태)
- 변경할 캐릭터 IP 이미지

**출력**:
- 새로운 캐릭터가 적용된 제품 목업 이미지 2장

**성공 기준**:
- 디자이너의 의도대로 목업화가 이루어졌는가
- 제품 기물의 물리적 구조가 유지되었는가
- 캐릭터의 실루엣과 비율이 유지되었는가

### 3.2 기능 B: 스케치 → 실사화
**설명**: 2D 드로잉을 실제 제품 사진처럼 변환

**입력**:
- 2D 스케치/드로잉 이미지
- (선택) 참조 질감/재질 이미지
- (선택) 참조 제품 형태 이미지

**출력**:
- 실사화된 제품 목업 이미지 2장

**성공 기준**:
- 디자인의 디테일 변화 없이 목업화가 이루어졌는가
- 캐릭터의 이목구비가 정확히 재현되었는가
- 지정된 질감/재질이 적용되었는가

### 3.3 기능 C: 히스토리 관리
**설명**: 마음에 드는 결과물을 저장하고 관리

**기능 상세**:
- 생성된 2장 중 1장 선택하여 히스토리에 저장
- 저장된 이미지를 언제든 불러와서 수정 가능
- 프로젝트 단위로 히스토리 관리
- 서버에서 삭제 및 다운로드 가능

### 3.4 기능 D: 부분 수정
**설명**: 결과물 이미지에서 특정 부분만 수정

**핵심 요구사항**:
- 사용자가 프롬프트로 특정 부분 수정 요청 가능
- **중요**: 요청한 특정 부분을 제외한 다른 곳은 절대 변하면 안 됨

**예시**:
- "아까 그거랑 똑같은데 재질만 바꿔줘"
- "색상만 빨간색으로 변경해줘"
- "캐릭터 위치만 조금 왼쪽으로"

## 4. 제품 카테고리

### 4.1 주요 카테고리 Top 3

| 순위 | 카테고리 | 예시 제품 |
|------|----------|-----------|
| 1 | 생활잡화 | 머그, 텀블러, 접시, 리빙잡화, 키링, 그립톡 |
| 2 | 봉제류 | 봉제인형, 인형키링, 담요, 쿠션 |
| 3 | 피규어류 | 피규어, 피규어 키링, 지비츠, 마그넷, 피규어가 들어간 머그/텀블러 |

### 4.2 특수 고려사항
- **투명 재질**: 투명한 재질의 제품을 이해하고 연출 가능해야 함
- **질감 다양성**: 플라스틱, 도자기, 금속, 봉제 등 다양한 질감 표현

## 5. 성공 지표 (품질 기준)

### 5.1 핵심 품질 지표

| 지표 | 설명 | 우선순위 |
|------|------|----------|
| **형태 유지력** | 입력한 더미의 형태와 구조를 정확히 유지 | 매우 높음 |
| **질감 유지력** | 사용자가 참조한 질감을 일관되게 유지 | 매우 높음 |
| **IP 일관성** | 캐릭터의 비율이 깨지지 않고 유지 | 매우 높음 |
| **재현성** | 동일 조건에서 동일한 결과 재생성 가능 | 매우 높음 |
| **투명 재질 처리** | 투명한 재질을 이해하고 정확히 연출 | 높음 |

### 5.2 절대 변경 불가 요소

- ❌ 캐릭터의 실루엣과 비율 (디포메이션 작업 제외)
- ❌ 캐릭터의 이목구비
- ❌ 제품 기물의 물리적 구조

### 5.3 참조 이미지 우선순위
- **1순위**: 구조 (Structure) 고정
- **2순위**: 스타일 (Style) 복사 (조건부 제어 가능해야 함)

## 6. 출력 요구사항

### 6.1 생성 사양

| 항목 | 요구사항 |
|------|----------|
| 생성 장수 | 1회 요청 시 2장 |
| 선택 | 2장 중 1장 선택 가능 |
| 기본 해상도 | 1K (API 출력 기본값) |
| 다운로드 해상도 | 2K 이상 (업스케일링) |
| 투명 배경 | 누끼 옵션 제공 필수 |

### 6.2 버전 관리

- 프로젝트 단위로 히스토리 관리
- 모든 생성 기록 저장
- 잘 나온 예시가 있으면 IP만 변경하여 재사용 가능

## 7. 기능 우선순위 (MoSCoW)

### Must Have (필수)
- [ ] 기능 A: 캐릭터 IP 변경
- [ ] 기능 B: 스케치 → 실사화
- [ ] 기능 C: 히스토리 관리 (저장/불러오기/삭제)
- [ ] 사용자 계정 관리 (이메일/비밀번호)
- [ ] 프로젝트 단위 관리
- [ ] 2장 생성 및 1장 선택
- [ ] 이미지 다운로드 (2K 업스케일)
- [ ] 투명 배경 옵션

### Should Have (권장)
- [ ] 기능 D: 부분 수정 (프롬프트 기반)
- [ ] 질감 참조 이미지 적용
- [ ] 구조 vs 스타일 우선순위 조절
- [ ] 생성 프롬프트 저장/재사용

### Could Have (선택)
- [ ] 배치 생성 (여러 IP 동시 적용)
- [ ] 결과물 비교 뷰
- [ ] 프롬프트 템플릿

### Won't Have (제외 - 1차 버전)
- [ ] 팀 협업 기능
- [ ] 외부 공유 링크
- [ ] 실시간 협업 편집

## 8. 사용 시나리오

### 8.1 시나리오 A: 신규 캐릭터 IP 적용
1. 디자이너가 프로젝트 생성 (예: "2026 봄 신상품")
2. 기존 인기 제품(다른 IP 적용) 이미지 업로드
3. 새로운 캐릭터 IP 이미지 업로드
4. "IP 변경" 모드 선택 후 생성
5. 2장 중 마음에 드는 1장 선택
6. 필요시 부분 수정 (색상/재질 변경)
7. 히스토리에 저장
8. 2K 해상도로 다운로드하여 기획 회의에 사용

### 8.2 시나리오 B: 스케치 제품화
1. 디자이너가 손으로 그린 머그컵 스케치 업로드
2. 참조할 재질 이미지 (도자기 질감) 업로드
3. "스케치 실사화" 모드 선택 후 생성
4. 2장 결과물 확인 후 1장 선택
5. "손잡이 색상만 검정으로 변경" 부분 수정 요청
6. 최종 결과물 저장 및 다운로드

## 9. 비기능 요구사항

### 9.1 성능
- 이미지 생성 시간: 30초 이내 (2장 기준)
- 업스케일링 시간: 10초 이내
- 동시 사용자: 5~20명 지원

### 9.2 신뢰성
- 생성 실패 시 자동 재시도 (최대 3회)
- 모든 히스토리 데이터 영구 보존

### 9.3 사용성
- 직관적인 UI (학습 없이 바로 사용 가능)
- 모던하고 깔끔한 디자인

### 9.4 보안
- 사용자 계정별 데이터 격리
- 이메일/비밀번호 인증
- 세션 기반 로그인 유지

## 10. 용어 정의

| 용어 | 정의 |
|------|------|
| IP (Intellectual Property) | 캐릭터 지적재산권, 여기서는 캐릭터 이미지/디자인을 의미 |
| 목업 (Mockup) | 실제 제품과 유사하게 제작된 시각적 모형/시뮬레이션 |
| 누끼 | 배경이 제거된 투명 배경 이미지 |
| 실사화 | 2D 이미지를 실제 제품처럼 3D 느낌으로 변환 |
| 디포메이션 | 캐릭터를 의도적으로 변형/축소하는 디자인 기법 |
| 기물 | 제품의 물리적 형태 (컵, 접시 등의 기본 구조) |

## 부록: 주요 질문과 답변

**Q: 한 번에 몇 장을 생성하나요?**
A: 1회 요청 시 2장을 생성하고, 그 중 1장을 선택합니다.

**Q: 투명 배경은 항상 적용되나요?**
A: 아니요, 사용자가 옵션으로 선택할 수 있습니다.

**Q: 팀원과 프로젝트를 공유할 수 있나요?**
A: 1차 버전에서는 개인 계정별 관리만 지원하며, 팀 기능은 추후 버전에서 추가 예정입니다.

**Q: 생성된 이미지는 얼마나 보관되나요?**
A: 사용자가 삭제하기 전까지 영구 보관됩니다.
