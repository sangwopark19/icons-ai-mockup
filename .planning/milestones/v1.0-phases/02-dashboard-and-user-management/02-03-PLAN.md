Data fetching:
```typescript
const fetchUsers = useCallback(async () => {
  if (!accessToken) return;
  setLoading(true);
  try {
    const res = await adminApi.listUsers(accessToken, { page: currentPage, ...searchParams });
    setUsers(res.data);
    setPagination(res.pagination);
  } catch (err) {
    console.error('Failed to fetch users:', err);
  } finally {
    setLoading(false);
  }
}, [accessToken, currentPage, searchParams]);

useEffect(() => { fetchUsers(); }, [fetchUsers]);
```

Action handlers:
- handleStatusChange: call adminApi.updateUserStatus, then refetch
- handleRoleChange: call adminApi.updateUserRole, then refetch
- handleDelete: call adminApi.softDeleteUser, then refetch
- Each handler wraps in try/catch, logs errors

Search handler:
- Receives { email, role, status } from UserSearchBar
- Sets searchParams state and resets currentPage to 1

Layout:
- Page title: "사용자 관리" as h1
- UserSearchBar below title
- UserTable below search bar
- Pass `user.id` from auth store as currentAdminId to prevent self-actions
