### Blocker

**Result/history UI does not display the model required by `PROV-03`.**

The API returns `provider` and `providerModel`, and Phase 10 intentionally maps product UI to `v1`/`v2` labels. That may be the desired product language, but it does not satisfy the literal requirement that the user can see which provider and model produced each generation.

Resolution options:

- Implement a product-safe model label in result/history, for example `v2 - GPT Image 2`, while keeping raw internal model strings out of normal UI.
- Or update `PROV-03` and traceability to state that users see product-safe provider version labels while model is retained in API/admin/support metadata.
