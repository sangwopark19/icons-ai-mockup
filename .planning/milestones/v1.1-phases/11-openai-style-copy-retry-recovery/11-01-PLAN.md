<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin API -> AdminService | Admin-only retry command crosses from authenticated HTTP route into service-layer recovery. Route authorization must remain unchanged. |
| Database Generation.promptData -> Queue payload | Persisted JSON is reconstructed into BullMQ job data. `copyTarget` and `selectedImageId` must be validated before queueing. |
| BullMQ job -> Worker dispatch | Queue payload is untrusted operational input. Worker must keep comparing queued fields against stored generation data before provider dispatch. |
| Worker -> Provider runtime | OpenAI and Gemini branches must stay isolated; OpenAI style-copy must not use Gemini `thoughtSignature`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | `AdminService.retryGeneration()` payload reconstruction | mitigate | Reconstruct `copyTarget` and `selectedImageId` only from persisted `generation.promptData`; do not accept retry request body metadata. |
| T-11-02 | Tampering | `copyTarget` JSON value | mitigate | Add `copyTargetValue()` literal guard accepting only `'ip-change'` or `'new-product'`; invalid stored values queue as absent and remain subject to worker mismatch rejection. |
| T-11-03 | Tampering | Worker stored/queued metadata guard | mitigate | Do not edit or relax `assertQueuedFieldMatchesStored()` checks for `copyTarget`, `selectedImageId`, `provider`, `providerModel`, or `styleReferenceId`; add regression proving complete payload passes without weakening mismatch tests. |
| T-11-04 | Information Disclosure | Cross-user selected style image | mitigate | Preserve worker style reference lookup and selected-image resolution; do not bypass `generationService.getById(userId, styleReferenceId)` or selected image lookup. |
| T-11-05 | Elevation of Privilege | Provider dispatch branch | mitigate | Preserve persisted `provider`/`providerModel` in retry payload and worker provider guard so OpenAI retries cannot fall into Gemini `thoughtSignature` runtime. |
| T-11-06 | Repudiation | Retry recovery observability | accept | Existing generation retryCount/status update remains the audit signal; Phase 11 does not add a new event log because the defect is deterministic metadata reconstruction and tests cover the payload. |
| T-11-07 | Denial of Service | Malformed stored continuation metadata | mitigate | Literal/string guards prevent malformed optional metadata from being trusted by admin retry; worker rejects mismatches before active-key lookup or vendor calls. |
</threat_model>
