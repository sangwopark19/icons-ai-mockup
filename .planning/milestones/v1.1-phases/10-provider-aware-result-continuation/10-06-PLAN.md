<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Query params -> browser state | `styleRef`, `copyTarget`, and `imageId` enter from URL and must be validated before use. |
| Browser -> upload/API | New target asset and optional instructions are sent to upload and generation APIs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-06-01 | Tampering | copyTarget query | mitigate | Accept only `ip-change` or `new-product`; show the required error copy otherwise. |
| T-10-06-02 | Information Disclosure | style reference image | mitigate | Fetch styleRef through authenticated API and require selected `imageId` from returned generation images. |
| T-10-06-03 | Provider Confusion | product UI | mitigate | Do not expose provider/model selectors; submit to the source generation copy-style endpoint. |
| T-10-06-04 | Prompt Overreach | user instructions | mitigate | Helper copy restricts instructions to replacement target; backend prompt wrappers enforce preserve constraints. |
| T-10-06-05 | Spoofing | unauthenticated style-copy access | mitigate | Redirect unauthenticated users to `/login` before fetching styleRef or submitting copy-style. |
| T-10-06-06 | Denial of Service | multi-click style-copy submit | mitigate | Guard `handleGenerate` with `!isGenerating` and disable submit while generation is in flight. |
</threat_model>
