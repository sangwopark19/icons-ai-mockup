# Phase 9 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — use the existing manual Next.js/Tailwind product UI |
| Preset | not applicable |
| Component library | local React components (`Button`, `Input`, `ImageUploader`) plus existing page-level cards, result panels, and history cards |
| Icon library | `lucide-react` for new utility/action icons; existing workflow emoji may remain where already used |
| Font | `Pretendard Variable`, `Pretendard`, system sans |

Source: `components.json` is absent, `apps/web/src/app/globals.css` defines the token system, and Phase 8 UI-SPEC locked the OpenAI workflow surfaces to the existing manual Tailwind UI. Do not initialize shadcn or add registry blocks in this phase.

User-facing product screens must not show `OpenAI`, `Gemini`, `GPT Image 2`, `gpt-image-2`, or raw provider/model values. Product workflow labels are `v1` and `v2`; internal records still store `provider = "openai"` and `providerModel = "gpt-image-2"`.

## Phase UI Scope

| Surface | Contract |
|---------|----------|
| `/projects/:id` project screen | Add a separate `스케치 실사화 v2` entry beside the existing `스케치 실사화 v1` entry. Keep existing `IP 변경 v1/v2` entries intact. |
| Project sidebar | Show workflow entries in this order: `IP 변경 v1`, `IP 변경 v2`, `스케치 실사화 v1`, `스케치 실사화 v2`, `히스토리`. |

| `/projects/:id/sketch-to-real/openai` | Add the v2 form route with required sketch input, optional texture/material reference, quality mode, preservation defaults, product category, material guidance, transparent-background option, and provider-hidden request wiring. |
| `/projects/:id/generations/:genId` | Generalize v2 result behavior to include `mode === "sketch_to_real"` with `provider === "openai"`: show a `v2` badge, exactly two candidates, candidate selection, save, download, and disabled guided follow-up actions. |
| `/projects/:id/history` | Show `v1`/`v2` badges on Sketch to Real cards as well as IP Change cards. Reopening a v2 Sketch to Real card must open the existing generation result page with v2 state preserved. |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner gaps, checkbox-to-label fine spacing |
| sm | 8px | Compact row gaps, chip gaps, candidate thumbnail gaps, inline button gap |
| md | 16px | Default element spacing, project card grid gap, header horizontal padding |
| lg | 24px | Form panel padding, card padding, result action panel padding |
| xl | 32px | Page vertical rhythm, upload-to-options spacing, result layout gap |
| 2xl | 48px | Empty state vertical padding, full-page loading spacing |
| 3xl | 64px | Page-level spacing only for full empty/loading states |

Exceptions: existing product cards may keep `rounded-xl` to match current project pages. New compact controls use `rounded-lg`. Icon-only controls must be at least `40px` on desktop and `44px` on mobile. Quality segments and category/material chips must keep a stable `min-height: 40px` desktop and `44px` mobile. Candidate thumbnails use a fixed square aspect ratio so labels, overlays, and selected states do not shift layout.

Use exactly these sizes and these two weights for new Phase 9 UI.

|------|------|--------|-------------|
| Label | 12px | 600 | 1.4 |
| Body | 14px | 400 | 1.5 |
| Heading | 18px | 600 | 1.2 |
| Display | 24px | 600 | 1.2 |

Labels include `v1`/`v2` badges, field labels, quality mode labels, category/material chip labels, candidate labels, helper text, and validation text. Body text includes form descriptions, result action copy, empty states, and toast/save messages. Heading is for page titles and panel titles. Display is for the project name only.

## Interaction Contract

| Interaction | Required behavior |
|-------------|-------------------|
| Project entry parity | `스케치 실사화 v1` and `스케치 실사화 v2` appear as sibling workflow entries, not a provider toggle. v1 remains reachable at `/projects/:id/sketch-to-real`; v2 uses `/projects/:id/sketch-to-real/openai`. |
| v2 visual emphasis | The v2 card has the same size and layout as v1, plus a subtle brand border or `v2` badge treatment. Do not use large marketing banners, hero sections, or copy that makes v1 look removed/deprecated. |
| Sidebar navigation | Both Sketch to Real links remain visible. The v2 link uses the same subtle brand-border treatment as the Phase 8 `IP 변경 v2` link. Current route gets the existing active/focus treatment if active state is implemented. |
| v2 form defaults | `quality = "medium"`, structure preservation ON, fixed viewpoint ON, fixed background ON. `transparentBackground` defaults OFF. User instructions and `기타` detail inputs default empty. |
| Required inputs | Disable the primary CTA until a sketch image, product category, and material guide are present. The texture image is optional. If `기타` is selected for category or material, its detail field is required before submit. |
| Sketch upload | Use the existing `ImageUploader` paste/file behavior. The sketch is the locked design spec; removing it clears preview state and disables submit. |
| Texture reference upload | The texture reference is optional and applies only to material, texture, finish, and color behavior. It must not imply a new product shape, character, logo, text, scene, prop, or background. Helper copy must not call it a style or scene reference. |
| Product category | Render presets as a dropdown or stable chip group. Category controls product structure together with the sketch. `기타` reveals a single-line free-text field below the control. |
| Material guidance | Render material presets as chips or a dropdown plus optional free text. If both material input and texture reference exist, the texture reference takes priority for material/finish while the material input disambiguates the request. |
| Quality mode | Render a three-option segmented radio control inside `생성 옵션`. Default selected value is `균형모드`; hidden request mapping is `빠른모드 -> low`, `균형모드 -> medium`, `퀄리티모드 -> high`. |
| Preservation controls | Show structure, viewpoint, and background preservation controls in the options panel. Defaults stay ON. User instructions cannot override layout, silhouette, proportions, face details, product construction, or perspective in Phase 9. |
| Transparent background | Keep `투명 배경 (누끼)` in the same option area as v1 for parity. The UI does not warn about model limitations; implementation generates opaque first and routes background removal/post-process behind the scenes. |
| User instructions | Instructions are additive inside the preservation contract. Do not change helper copy to imply freeform redesign, new props, new logos, or new scene creation. |
| Submit behavior | On submit, upload sketch and optional texture, create an OpenAI-backed v2 `sketch_to_real` generation with `outputCount: 2`, then navigate to `/projects/:id/generations/:genId`. |
| Loading/progress | Result page polls using the existing 2-second pattern. Pending/processing states show the v2 loading copy and no blank intermediate canvas. Keep spinner styling consistent with existing `border-brand-500` loaders. |
| Candidate layout | Completed v2 Sketch to Real results must render exactly two candidates. The selected large image remains the primary focal area; the two thumbnails sit in a stable square grid with `후보 1`/`후보 2` labels and a `선택됨` overlay. |
| Candidate selection | Clicking a candidate updates local selected state immediately and persists selection through the existing `/select` endpoint. Selection state must survive save, reopen, and download. |
| Save lifecycle | `히스토리에 저장` saves the currently selected candidate through the existing image save lifecycle. Show success/error message near the action stack for 3 seconds; do not navigate away automatically. |
| History reopen | v2 history cards open the same result route. The result page derives the `v2` label from stored provider/version data and keeps unsupported follow-up actions disabled. |
| Download lifecycle | `선택 이미지 다운로드` downloads the selected candidate only. If no candidate is selected, the control is disabled rather than downloading the first image silently. |
| Unsupported v2 follow-ups | For v2 in Phase 9, `부분 수정`, both `스타일 복사` actions, and `동일 조건 재생성` must not execute. Render them disabled with the guided copy `v2 후속 편집은 다음 업데이트에서 지원됩니다`. |
| Condition edit | `조건 수정` may remain enabled for v2, but it must route back to `/projects/:id/sketch-to-real/openai`, not the v1 route. |
| Failure state | Failed v2 Sketch to Real generations show the error copy above, the backend message only if it is already Korean/actionable, and a `다시 시도` action that returns to `/projects/:id/sketch-to-real/openai`. |

## Responsive Behavior

| Breakpoint | Contract |
|------------|----------|
| Mobile `< 640px` | Project workflow cards stack in one column. Sidebar navigation must not cause horizontal overflow; if the surrounding page is touched, convert it to a top stacked nav or horizontal scroll area. Uploaders stack sketch first, texture reference second. Category/material controls wrap without truncating labels. |
| Tablet `640px-1023px` | Workflow cards use two columns where space allows. The Sketch to Real v1/v2 pair must remain adjacent after the IP Change v1/v2 pair. Result page stacks selected image above candidate/action panel. |
| Desktop `>= 1024px` | Form uses the existing two-column uploader layout. Options remain in one panel below uploaders. Result page uses `1fr / 300px` or equivalent, with selected image left and candidates/actions right. |

No text may overflow buttons, badges, chips, cards, or thumbnails at 360px width. Long project names, `기타` entries, and material descriptions wrap. Provider/model internals remain hidden on product screens.

## Accessibility Contract

| Area | Requirement |
|------|-------------|
| Version labels | `v1`/`v2` badges must appear as text, not color-only indicators. |
| Quality mode | Use a keyboard-operable `radiogroup` or native radio inputs. Selected state must be exposed with `aria-checked` or native checked state. |
| Category/material controls | Chip groups use native radio inputs or buttons with `aria-pressed`. Dropdowns use native `select` or accessible combobox semantics. `기타` detail fields must be programmatically associated with their labels. |
| Uploaders | Preserve existing paste and file-select keyboard access. Image remove buttons need Korean `aria-label` values: `스케치 이미지 제거`, `재질 참조 이미지 제거`. |
| Texture reference explanation | The material-only constraint must be visible helper text near the texture uploader, not hidden only in placeholder text. |
| Candidate thumbnails | Buttons need `aria-label` values: `후보 1 선택`, `후보 2 선택`. Selected candidate exposes visible `선택됨` text and `aria-pressed=true` or equivalent selected state. |
| Disabled follow-ups | Disabled v2 follow-up buttons need `aria-disabled`/`disabled` and `aria-describedby` pointing to the guided copy. |
| Download/save | Buttons remain reachable by keyboard and show loading/disabled state while requests are in flight. |
| Focus | Use existing `focus-visible` brand ring. Do not remove outlines from links, segmented controls, chips, thumbnails, upload controls, or icon buttons. |
| Images | Result images use descriptive alt text: `선택된 v2 스케치 실사화 결과`, `v2 스케치 실사화 후보 1`, `v2 스케치 실사화 후보 2`. |
