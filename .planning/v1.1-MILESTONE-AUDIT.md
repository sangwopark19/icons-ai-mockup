### Blocker

**Admin retry drops OpenAI style-copy continuation metadata**

Affected requirements: `OPS-03`, `OED-02`, `OED-03`

Phase 10 added `copyTarget` and `selectedImageId` as required continuation metadata for OpenAI style-copy. Normal user flow persists and queues these fields. The Phase 7 admin retry path requeues failed generations without those fields, while the worker validates queued values against stored prompt data before provider execution.

Impact: a failed OpenAI style-copy generation can be created normally, but admin retry can requeue an incomplete job and fail before vendor execution. This breaks the admin recovery flow for OpenAI style-copy follow-ups.

Suggested fix:

- Include `copyTarget` and `selectedImageId` from persisted `promptData` in `AdminService.retryGeneration()` queue payload.
- Add a regression test for failed OpenAI style-copy retry.
