### DASH-04 Gap Detail

**Requirement:** "활성 Gemini API 키 정보 및 호출 횟수 표시"

During Phase 2, DASH-04 was implemented with a placeholder (`value="N/A"`, `placeholder={true}`) because the ApiKey model didn't exist yet. Phase 4 completed the backend — `getDashboardStats()` now queries `prisma.apiKey.findFirst({ where: { isActive: true } })` and returns `activeApiKeys: { alias, callCount } | null`. However, two frontend files were never updated:

1. **`apps/web/src/lib/api.ts` line 195:** `DashboardStats.activeApiKeys` typed as literal `null` instead of `{ alias: string; callCount: number } | null`
2. **`apps/web/src/app/admin/dashboard/page.tsx` lines 100-105:** KPI card hardcodes `value="N/A"` unconditionally, never reads `stats.activeApiKeys`

**Fix required (2 files, ~10 lines):**
- Update `DashboardStats` type in `api.ts`
- Update KPI card render in `dashboard/page.tsx` to conditionally display alias + call count

### KEY-06 Resolution Note

04-VERIFICATION.md flagged KEY-06 as partial (edit.routes.ts missing `incrementCallCount`). This was fixed in commit `a6ae466` — all 4 Gemini call paths now increment the call count correctly.
