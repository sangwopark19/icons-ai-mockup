### KEY-06 Gap Detail

**Requirement:** "각 API 키의 호출 횟수가 표시된다"
**Phase:** 4 (API Key Management)

`edit.routes.ts` calls `adminService.getActiveApiKey()` but only destructures `key` — not `id`. Therefore `incrementCallCount()` is never called for edit-mode Gemini API calls. Worker correctly increments for all 3 other paths (ip_change, sketch_to_real, style_copy).

**Impact:** Call count in admin UI under-counts actual API usage when edit operations are performed.

**Fix (2 lines in `edit.routes.ts`):**
1. `const { key: activeApiKey }` → `const { id: activeKeyId, key: activeApiKey }`
2. Add `await adminService.incrementCallCount(activeKeyId);` before `geminiService.generateEdit()`
