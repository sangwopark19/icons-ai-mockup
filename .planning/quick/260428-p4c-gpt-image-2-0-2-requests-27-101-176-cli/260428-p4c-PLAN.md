<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser -> Fastify API | Authenticated user input creates generation jobs and could double-submit or poll aggressively. |
| worker -> OpenAI API | Uploaded product/reference images and prompts leave the app boundary; hidden retries can multiply cost. |
| API/admin UI -> operator | Support metadata is useful but must not expose API keys, raw providerTrace, base64 images, or raw vendor payloads. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-OPENAI-USAGE-01 | I | REQUEST-AUDIT.md | mitigate | Store only counts, timestamps, generation IDs, job IDs, and request IDs; no secrets or image bytes. |
| T-OPENAI-USAGE-02 | D | worker/OpenAI API | mitigate | Disable hidden OpenAI SDK retries and OpenAI BullMQ vendor replays so one user job has a bounded request budget. |
| T-OPENAI-USAGE-03 | R | admin usage display | mitigate | Persist and render safe request-accounting fields that distinguish app calls, candidates, queue attempts, and SDK retries. |
| T-OPENAI-USAGE-04 | I | admin detail modal | mitigate | Expose selected safe counts/IDs only; never render raw providerTrace, encrypted keys, or raw vendor response bodies. |
</threat_model>
