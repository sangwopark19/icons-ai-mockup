Locked handoff facts:
- Latest production `gpt-image-2` failures are local OpenAI SDK `APIConnectionTimeoutError` after about 60 seconds, with no OpenAI status/code/requestID.
- `apps/api/src/services/openai-image.service.ts` currently constructs OpenAI clients with `timeout: 60_000` and `sdkMaxRetries = 0`.
- Completed OpenAI `ip_change` medium jobs previously took about 243.9s and 560.1s, so 60s is too low.
- Queue attempts for OpenAI jobs are intentionally `1` to avoid duplicate vendor calls/cost. Do not blindly re-enable retries.
- Required fix: configurable `OPENAI_IMAGE_TIMEOUT_MS`, default around `600000` ms, while keeping retry behavior deliberate.
- UI request: fast, balanced, and quality modes must each show official-documentation-based latency guidance. OpenAI official docs do not publish minute-by-minute latency estimates by quality mode.
