# Quick Task 260430-k39 Summary

OpenAI GPT Image 2 SDK timeout을 `OPENAI_IMAGE_TIMEOUT_MS`로 설정 가능하게 만들고 기본값을 `600_000`ms로 올렸습니다. SDK retry budget은 `maxRetries: 0`으로 유지했습니다. OpenAI 공식 문서가 모드별 분 단위 대기시간을 공개하지 않으므로, UI의 임의 분 단위 범위를 제거하고 공식 문서 기반의 상대 지연 안내로 정정했습니다.

## Changes

- `apps/api/src/services/openai-image.service.ts`
  - `OPENAI_IMAGE_TIMEOUT_MS` resolver 추가
  - empty, non-numeric, zero, negative env 값은 `600_000`으로 fallback
  - 다섯 개 OpenAI client 생성 지점의 hardcoded `60_000` timeout 제거
- `apps/api/src/services/__tests__/openai-image.service.test.ts`

## Official Documentation Correction

- OpenAI Image Generation guide states `low` quality is the fastest option.
- The same guide positions `medium`/`high` as higher-quality choices for final assets.
- The official docs do not publish minute-by-minute latency estimates for `low`, `medium`, and `high`.
- UI copy now avoids unsupported exact ranges and says requests can vary by content and image count.
