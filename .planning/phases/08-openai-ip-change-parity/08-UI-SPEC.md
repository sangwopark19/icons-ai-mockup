# Phase 8 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — use the existing manual Next.js/Tailwind product UI |
| Preset | not applicable |
| Component library | local React components (`Button`, `Input`, `ImageUploader`) plus existing page-level cards/modals |
| Icon library | `lucide-react` for new utility/action icons; existing workflow emoji may remain where already used |
| Font | `Pretendard Variable`, `Pretendard`, system sans |

Source: `components.json` is absent, `apps/web/src/app/globals.css` defines the token system, and Phase 7 UI-SPEC explicitly kept the project on local Tailwind components. Do not initialize shadcn or add registry blocks in this phase.

## Phase UI Scope

| Surface | Contract |
|---------|----------|
| `/projects/:id` project screen | Add a separate `IP 변경 v2` entry beside the existing `IP 변경 v1` entry. Keep the existing Gemini route unchanged at `/projects/:id/ip-change`; the new route is `/projects/:id/ip-change/openai`. |
| Project sidebar | Show both IP Change entries in the same project context. Order: `IP 변경 v1`, `IP 변경 v2`, then existing `스케치 실사화` and `히스토리`. |
| `/projects/:id/ip-change` | Preserve existing v1 behavior and route. If touched, label the page as `IP 변경 v1` without changing request defaults. |
| `/projects/:id/ip-change/openai` | Reuse the current IP Change form surface with v2 defaults, quality mode, provider-hidden request wiring, and the same upload/options semantics. |
| `/projects/:id/generations/:genId` | For v2 generations, show a `v2` badge, exactly two candidates, candidate selection, save to history, download, and disabled guided follow-up actions. |
| `/projects/:id/history` | Show `v1`/`v2` badges on IP Change cards. Reopening a v2 card must open the existing generation result page with v2 state preserved. |

User-facing product screens must not show `OpenAI`, `Gemini`, `GPT Image 2`, `gpt-image-2`, or raw provider/model values for this phase. Internal records still store `provider = "openai"` and `providerModel = "gpt-image-2"`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner gaps, checkbox-to-label fine spacing |
| sm | 8px | Compact row gaps, candidate thumbnail gaps, inline button gap |
| md | 16px | Default element spacing, project card grid gap, header horizontal padding |
| lg | 24px | Form panel padding, card padding, result action panel padding |
| xl | 32px | Page vertical rhythm, form-to-options spacing, result layout gap |
| 2xl | 48px | Empty state vertical padding, full-page loading spacing |
| 3xl | 64px | Page-level spacing only for full empty/loading states |

Exceptions: existing product cards may keep `rounded-xl` to match the current project pages. New compact controls use `rounded-lg`. Icon-only controls must be at least `40px` on desktop and `44px` on mobile. Candidate thumbnails use a fixed square aspect ratio so selection overlays do not shift layout.

## Typography

Use exactly these sizes and weights for new Phase 8 UI.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px | 600 | 1.4 |
| Body | 14px | 400 | 1.5 |
| Heading | 18px | 600 | 1.2 |
| Display | 24px | 600 | 1.2 |

Labels include `v1`/`v2` badges, field labels, quality mode labels, candidate labels, and helper/error text. Body text includes form descriptions, result action copy, empty states, and toast/save messages. Heading is for page titles and panel titles. Display is for the project name only.

| Secondary (30%) | `#141416` (`--bg-secondary`) and `#1C1C1F` (`--bg-tertiary`) | Sidebar, workflow cards, form panels, upload zones, result panels, history cards |

## Interaction Contract

| Interaction | Required behavior |
|-------------|-------------------|
| Project entry parity | v1 and v2 appear as sibling workflow entries, not a provider toggle that replaces one route with another. v1 remains reachable at `/projects/:id/ip-change`; v2 uses `/projects/:id/ip-change/openai`. |
| v2 visual emphasis | The v2 card has the same size and layout as v1, plus a subtle brand border or `v2` badge treatment. Do not use large marketing banners, hero sections, or copy that makes v1 look removed/deprecated. |
| Sidebar navigation | Both IP Change links remain visible. Current route gets the existing active/focus treatment if active state is implemented; otherwise keep hover/focus consistent with existing links. |
| v2 form defaults | `preserveStructure`, `fixedViewpoint`, and `fixedBackground` are enabled by default. `preserveHardware`, `transparentBackground`, and `removeShadows` default to off. `userInstructions` and hardware details default empty. |
| Quality mode | Render a three-option segmented radio control inside `생성 옵션`. Default selected value is `균형모드`; mapping is hidden in code: `빠른모드 -> low`, `균형모드 -> medium`, `퀄리티모드 -> high`. |
| Transparent background | Keep the `투명 배경 (누끼)` option for parity. The UI does not warn about model limitations; implementation generates opaque first and routes background removal/post-process behind the scenes. |
| User instructions | Instructions are additive inside the preservation contract. Do not change helper copy to imply freeform structure redesign unless the user explicitly writes override language. |
| Submit behavior | Primary CTA is disabled until both required images are present. On submit, upload both images, create an OpenAI-backed v2 generation, and navigate to `/projects/:id/generations/:genId`. |
| Loading/progress | Result page polls using the existing 2-second pattern. Pending/processing states show the v2 loading copy and no blank intermediate canvas. Keep spinner styling consistent with existing `border-brand-500` loaders. |
| Candidate layout | Completed v2 results must render exactly two candidates. The selected large image remains the primary focal area; the two thumbnails sit in a stable square grid with `후보 1`/`후보 2` labels and a `선택됨` overlay. |
| Candidate selection | Clicking a candidate updates local selected state immediately and persists selection through the existing `/select` endpoint. Selection state must survive save, reopen, and download. |
| Save lifecycle | `히스토리에 저장` saves the currently selected candidate through the existing image save lifecycle. Show success/error message near the action stack for 3 seconds; do not navigate away automatically. |
| History reopen | v2 history cards open the same result route. The result page derives the `v2` label from stored provider/version data and keeps unsupported follow-up actions disabled. |
| Download lifecycle | `다운로드` downloads the selected candidate only. If no candidate is selected, the control is disabled rather than downloading the first image silently. |
| Unsupported v2 follow-ups | For v2 in Phase 8, `부분 수정`, both `스타일 복사` actions, and `동일 조건 재생성` must not execute. Render them disabled with the guided copy `v2 후속 편집은 다음 업데이트에서 지원됩니다`. |
| Condition edit | `조건 수정` may remain enabled for v2, but it must route back to `/projects/:id/ip-change/openai`, not the v1 route. |
| Failure state | Failed v2 generations show the error copy above, the backend message only if it is already Korean/actionable, and a `다시 시도`/back action that returns to the v2 form context. |

## Responsive Behavior

| Breakpoint | Contract |
|------------|----------|
| Mobile `< 640px` | Project workflow cards stack in one column. Sidebar navigation must not cause horizontal overflow; if the surrounding page is touched, convert it to a top stacked nav or horizontal scroll area. Form uploads stack source image first, character image second. |
| Tablet `640px-1023px` | Workflow cards use two columns where space allows. The v1/v2 pair must remain adjacent before `스케치 실사화`. Result page stacks selected image above candidate/action panel. |
| Desktop `>= 1024px` | Form uses the existing two-column uploader layout. Result page uses `1fr / 300px` or equivalent, with selected image left and candidates/actions right. |

No text may overflow buttons, badges, cards, or thumbnails at 360px width. Long project names and character names wrap; provider/model internals remain hidden on product screens.

## Accessibility Contract

| Area | Requirement |
|------|-------------|
| Version labels | `v1`/`v2` badges must appear as text, not color-only indicators. |
| Quality mode | Use a keyboard-operable `radiogroup` or native radio inputs. Selected state must be exposed with `aria-checked` or native checked state. |
| Uploaders | Preserve existing paste and file-select keyboard access. Image remove buttons need Korean `aria-label` such as `원본 제품 이미지 제거`. |
| Candidate thumbnails | Buttons need `aria-label` values: `후보 1 선택`, `후보 2 선택`. Selected candidate exposes visible `선택됨` text and `aria-pressed=true` or equivalent selected state. |
| Disabled follow-ups | Disabled v2 follow-up buttons need `aria-disabled`/`disabled` and `aria-describedby` pointing to the guided copy. |
| Download/save | Buttons remain reachable by keyboard and show loading/disabled state while requests are in flight. |
| Focus | Use existing `focus-visible` brand ring. Do not remove outlines from links, segmented controls, thumbnails, or icon buttons. |
| Images | Result images use descriptive alt text: `선택된 v2 IP 변경 결과`, `v2 IP 변경 후보 1`, `v2 IP 변경 후보 2`. |
