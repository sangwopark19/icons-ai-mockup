# Plan 08-03: V2 Result And History Lifecycle

<objective>
Make OpenAI IP Change outputs complete the existing candidate selection, save, history reopen, and download lifecycle while showing only v1/v2 labels and disabling Phase 10 follow-up actions for v2.
</objective>

<threat_model>
## Threats

| ID | Severity | Threat | Mitigation |
|----|----------|--------|------------|
| T-08-08 | high | v2 users trigger partial edit, style copy, or same-condition regeneration before OpenAI lineage work exists | Disable those controls for v2 with `v2 후속 편집은 다음 업데이트에서 지원됩니다`. |
| T-08-09 | medium | Result/history UI leaks raw provider/model values | Map `provider === "openai"` to `v2` and all other IP Change generations to `v1`. |
| T-08-10 | medium | Download or save silently uses the wrong candidate | Disable download when no selected candidate exists; keep selection persisted through `/select`. |
</threat_model>
