# Plan 08-01: Backend OpenAI IP Change Runtime

<objective>
Add the backend OpenAI GPT Image 2 IP Change runtime while preserving existing Gemini worker behavior. OpenAI `provider = "openai"` and `providerModel = "gpt-image-2"` jobs for `mode = "ip_change"` must produce exactly two output candidates, capture support metadata, and never send forbidden GPT Image 2 parameters.
</objective>

<threat_model>
## Threats

| ID | Severity | Threat | Mitigation |
|----|----------|--------|------------|
| T-08-01 | high | OpenAI API key, uploaded image bytes, or raw vendor response leaks into logs/product UI | Use provider-scoped `adminService.getActiveApiKey("openai")`; log only request/support IDs and error messages; do not persist raw base64 or raw response bodies. |
| T-08-02 | high | Queue/provider mismatch dispatches OpenAI jobs into Gemini or Gemini jobs into OpenAI | Preserve existing persisted-vs-queued `provider` and `providerModel` checks before dispatch. |
| T-08-03 | medium | Unsupported GPT Image 2 parameters cause runtime failures | Unit test that `background: "transparent"` and `input_fidelity` are absent from `client.images.edit()` calls. |
| T-08-04 | medium | OpenAI non-IP workflows accidentally execute early | Keep `provider === "openai"` blocked for non-`ip_change` modes with an explicit Korean error. |
</threat_model>

    Create `apps/api/src/services/openai-image.service.ts` as a separate service beside `gemini.service.ts`.

    Implement:
    - `export type OpenAIQuality = 'low' | 'medium' | 'high'`
    - `export interface OpenAIIPChangeOptions` with `preserveStructure`, `transparentBackground`, `preserveHardware`, `fixedBackground`, `fixedViewpoint`, `removeShadows`, `userInstructions`, `hardwareSpecInput`, `hardwareSpecs`, `quality`, and `prompt`.
    - `export interface OpenAIImageGenerationResult` with `images: Buffer[]`, `requestIds: string[]`, `responseId?: string`, `imageCallIds: string[]`, `revisedPrompt?: string`, and `providerTrace: Record<string, unknown>`.
    - `generateIPChange(apiKey, sourceImageBase64, characterImageBase64, options): Promise<OpenAIImageGenerationResult>`.

    Use the official SDK and Image API edit path:
    - instantiate `new OpenAI({ apiKey, maxRetries: 2, timeout: 60_000 })`.
    - call `client.images.edit()` with `model: 'gpt-image-2'`.
    - use the source product image and character reference image as separate input images.
    - default `quality` to `'medium'`.
    - produce exactly two candidate buffers with `n: 2`.

    Build a structured prompt with exact section labels:
    - `Task:`
    - `Image roles:`
    - `Must change:`
    - `Must preserve:`
    - `Hard constraints:`
    - `Output:`

    Include exact prompt requirements:
    - `Edit Image 1 by replacing only the existing character/IP artwork with the character from Image 2.`
    - preserve product geometry, dimensions, crop, camera viewpoint, perspective, material, lighting, hardware, label placement, non-character text, and non-target areas.
    - preserve Image 2 character silhouette, proportions, face details, colors, and recognizable motifs.
    - never add extra characters, logos, watermark, text, props, accessories, or decorative effects.

    Do not send `background: "transparent"` to `gpt-image-2`. Do not send `input_fidelity`.
