# Phase 7 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — use the existing manual Next.js/Tailwind admin UI |
| Preset | not applicable |
| Component library | local React components (`Button`, `Input`, `ConfirmDialog`, admin tables/modals) |
| Icon library | `lucide-react` |

Source: `components.json` is absent, `apps/web/src/app/globals.css` defines the token system, and prior admin phases use local components rather than shadcn. Do not initialize shadcn or add registry blocks in this phase.

## Phase UI Scope

| Surface | Contract |
|---------|----------|
| `/admin/api-keys` | Replace the single global key table with provider tabs: `Gemini` and `OpenAI`. Each tab owns its own key list, active key state, empty state, add modal, activation confirmation, and delete confirmation. |
| `/admin/dashboard` | Do not show a single global active key. Display provider-scoped active key information so Gemini and OpenAI cannot be mistaken for one shared runtime key. |
| `/admin/content` generation table | Add provider/model visibility to generation monitoring. Use compact provider badges and show model as text metadata. |
| Generation detail modal | Add a safe support metadata section with provider, model, OpenAI request ID, response ID, image call ID, and revised prompt. Keep raw API keys, uploaded image bytes/base64, and raw vendor responses out of the UI. |
| Retry controls | Keep the existing `재시도` label, but retry must preserve the failed generation's saved provider/model. Loading and error states must make wrong-provider failures visible. |

Source: Phase 7 CONTEXT D-06 through D-19, REQUIREMENTS `OPS-01` through `OPS-04`, and research summary "provider/schema/admin foundation".

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge inner gaps, copy-icon spacing |
| sm | 8px | Compact row gaps, tab gap, inline button gap |
| md | 16px | Default element spacing, table cell horizontal padding, modal form gaps |
| lg | 24px | Modal padding, page section gaps |
| xl | 32px | Admin page vertical spacing, major content groups |
| 2xl | 48px | Empty state vertical padding |
| 3xl | 64px | Page-level spacing only when a full empty panel needs breathing room |

Exceptions: table cells may use `12px` vertical padding (`py-3`) to match existing admin tables. Icon-only controls must be at least `40px` on desktop and `44px` on mobile.

## Typography

Use exactly these sizes and weights for new Phase 7 UI.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px | 600 | 1.4 |
| Body | 14px | 400 | 1.5 |
| Heading | 18px | 600 | 1.2 |
| Display | 24px | 600 | 1.2 |

Labels include table headers, field labels, tab count badges, and metadata labels. Body text includes table rows, modal body copy, helper/error text, and toast copy. Heading is for modal titles and section titles. Display is for page titles such as `API 키 관리`.

## Copywriting Contract

All user-facing admin copy in this phase uses Korean, except provider/model values and vendor identifiers.

| Element | Copy |
|---------|------|
| Primary CTA | `{Provider} 키 추가` |
| Empty state heading | `등록된 {Provider} API 키가 없습니다` |
| Empty state body | `{Provider} 생성 작업을 실행하려면 API 키를 추가하고 활성화하세요.` |
| Error state | `API 키 목록을 불러오지 못했습니다. 잠시 후 다시 시도하거나 API 로그를 확인해주세요.` |
| Destructive confirmation | `API 키 삭제`: `키 [{alias}]을 삭제합니다.` |

Additional required copy:

| Element | Copy |
|---------|------|
| Provider tabs | `Gemini`, `OpenAI` |
| Add key modal title | `{Provider} API 키 추가` |
| Add key success toast | `{Provider} API 키 '{alias}'가 추가되었습니다` |
| Activate confirmation | `활성 {Provider} 키를 [{alias}]으로 전환합니다. 새 {Provider} 생성 작업부터 이 키를 사용합니다.` |
| Activate success toast | `{Provider} API 키 '{alias}'가 활성화되었습니다` |
| Delete success toast | `{Provider} API 키 '{alias}'가 삭제되었습니다` |
| Active key blocked delete | Active key rows do not render a delete button. If backend rejects deletion, show `활성 API 키는 삭제할 수 없습니다. 다른 키를 활성화한 뒤 삭제해주세요.` |
| Missing active key error | `{Provider} API 키가 설정되지 않았습니다` |
| Provider mismatch error | `저장된 provider와 큐 provider가 일치하지 않아 작업을 실행할 수 없습니다.` |
| Generation empty state | `생성 작업이 없습니다` |
| Generation detail support section | `지원 정보` |
| Copy identifier success toast | `식별자가 복사되었습니다` |

Destructive actions in this phase:

| Action | Confirmation approach |
|--------|-----------------------|
| Delete inactive API key | Use existing `ConfirmDialog`, danger variant, copy above, `삭제` confirm label. |

Non-destructive but state-changing actions:

| Action | Confirmation approach |
|--------|-----------------------|
| Activate provider key | Use existing `ConfirmDialog`, default variant, provider-scoped copy above, `활성화` confirm label. |
| Retry failed generation | No confirmation dialog. Button shows loading, preserves provider/model, refreshes row status to `대기중` on success, and shows provider mismatch errors if routing validation fails. |

## Interaction Contract

| Interaction | Required behavior |
|-------------|-------------------|
| Provider tabs | Use a tablist/segmented-control pattern directly under the page header. `Gemini` is the default selected tab after v1.0 migration. Tabs must not share active-key state. |
| Add key modal | Modal inherits provider from the selected tab. Do not use a free provider text input. Title, submit CTA, and toast include provider. API key input remains password-masked. |
| Key table | Columns remain: 별칭, 마스킹된 키, 상태, 호출 횟수, 등록일, 마지막 사용일, 액션. In provider tabs, omit a provider column to avoid redundant noise. |
| Active key row | Shows `활성` badge. Does not show `삭제` or `활성화`. |
| Inactive key row | Shows `비활성` badge plus `활성화` and `삭제`. |
| Dashboard active keys | Show Gemini and OpenAI as separate compact rows or cards. Missing provider key displays `활성 키 미설정`, without implying the other provider is also missing. |
| Generation table | Add `Provider` and `Model` columns or a compact stacked provider/model cell. Provider badge text must be `Gemini` or `OpenAI`; model text must preserve exact stored value such as `gemini-3-pro-image-preview` or `gpt-image-2`. |
| Generation detail support info | Use a collapsed-by-default section when identifiers are present. Each identifier row has label, monospace value, and a copy icon button with `aria-label`. `openaiRevisedPrompt` may render as wrapped text. |
| Trace safety | Never render raw API keys, uploaded image bytes/base64, raw vendor responses, or full `providerTrace` JSON by default. |
| Loading | Preserve existing table spinner/`불러오는 중...` patterns. Provider tab changes show loading only inside the table area, not a full-page block. |
| Toasts | Keep fixed top-right toast. Use success/error color only; no new toast library. |
| Keyboard/accessibility | Tabs expose selected state; modals close with Escape and stop propagation on modal body click; copy buttons and icon-only buttons have Korean `aria-label`. |
