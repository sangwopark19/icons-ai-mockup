# Plan 09-04: Phase 9 Smoke And Release Verification

<objective>
Create the Phase 9 smoke checklist and run final automated, browser, and gated real-OpenAI verification for the OpenAI Sketch to Real v2 rollout.
</objective>

<threat_model>
## Threats

| ID | Severity | Threat | Mitigation |
|----|----------|--------|------------|
| T-09-15 | high | Real OpenAI smoke exposes API key or sample images in committed files | Keep `OPENAI_API_KEY` in environment only; do not commit sample inputs/outputs unless explicitly sanitized. |
| T-09-16 | medium | Automated checks pass but real GPT Image 2 rejects multi-image Sketch edit request shape | Gate real smoke on `OPENAI_API_KEY`; document request IDs, input roles, output count, and forbidden-parameter checks. |
| T-09-17 | medium | Product UI visually regresses on mobile or leaks provider names | Include browser checklist for project, v2 form, result, history, and 360px width. |
| T-09-18 | medium | Transparent option appears to work but final file is opaque or visibly has a white box/halo | Include alpha-channel, transparent-pixel ratio, border transparency, and dark-background composite evidence. |
| T-09-21 | medium | Candidate order changes after reload/history reopen | Include smoke steps that select candidate 2, reload, save, reopen from history, and verify the same 후보 1/후보 2 order. |
</threat_model>
