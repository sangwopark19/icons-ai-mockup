# Plan 09-03: V2 Sketch Result And History Lifecycle

<objective>
Generalize the existing v2 result and history lifecycle so OpenAI `sketch_to_real` results show v2 state, exactly two candidates, candidate selection, save, download, history reopen, Sketch-specific retry/condition-edit routing, and disabled Phase 10 follow-up actions.
</objective>

<threat_model>
## Threats

| ID | Severity | Threat | Mitigation |
|----|----------|--------|------------|
| T-09-11 | medium | Result page treats OpenAI Sketch as v1 because v2 detection is `ip_change`-only | Compute v2 from `generation.provider === "openai"` and mode-specific workflow copy. |
| T-09-12 | medium | Condition edit sends v2 Sketch users back to Gemini v1 route | Route OpenAI Sketch condition edit and failed retry to `/projects/:id/sketch-to-real/openai`. |
| T-09-13 | medium | Phase 10 actions execute early and mix OpenAI state with Gemini-only follow-up logic | Disable partial edit, style copy, and same-condition regeneration for all OpenAI v2 results with guided copy. |
| T-09-14 | low | History omits v2 badge for Sketch and makes saved results look like Gemini | Render v1/v2 badge for Sketch to Real as well as IP Change. |
</threat_model>
