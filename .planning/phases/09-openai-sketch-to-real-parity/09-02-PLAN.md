# Plan 09-02: V2 Project Entry And Sketch Form

<objective>
Expose `스케치 실사화 v2` beside the existing `스케치 실사화 v1` entry and add a v2 form route that submits OpenAI Sketch to Real requests with required sketch input, optional texture/material reference, product category, material guidance, preservation defaults, quality mapping, two candidate output, and transparent-background option.
</objective>

<threat_model>
## Threats

| ID | Severity | Threat | Mitigation |
|----|----------|--------|------------|
| T-09-07 | medium | Product UI leaks provider/model names hidden by Phase 9 | Render only `v1`, `v2`, `스케치 실사화 v1`, and `스케치 실사화 v2`; keep provider/model values only in request payload. |
| T-09-08 | high | V2 form accidentally submits to Gemini by omitting provider fields | Submit exact `provider: "openai"` and `providerModel: "gpt-image-2"` in the v2 create body. |
| T-09-09 | medium | V1 route behavior changes while adding v2 | Keep `/projects/:id/sketch-to-real` and its Gemini-default create payload unchanged except optional v1 label copy. |
| T-09-10 | medium | Invalid `기타` inputs or missing product/material guidance create vague prompts | Disable submit until sketch, category, and material guidance are present; require detail text when `기타` is selected. |
| T-09-19 | medium | OpenAI edit request fails after upload because input files are unsupported | Validate v2 sketch and texture files client-side as PNG/JPEG only and max 10MB before upload/create request. |
</threat_model>
