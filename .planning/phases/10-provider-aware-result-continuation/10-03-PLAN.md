<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> edit route | User prompt and selected image ID cross into authenticated API code. |
| File storage -> OpenAI | Selected generated image bytes are sent to OpenAI for edit. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03-01 | Tampering | selected image resolution | mitigate | Resolve `selectedImageId` only inside `generation.images` returned by `generationService.getById(user.id, generationId)`. |
| T-10-03-02 | Elevation of Privilege | provider routing | mitigate | Branch from persisted `generation.provider`; OpenAI failures never call Gemini. |
| T-10-03-03 | Information Disclosure | OpenAI metadata | mitigate | Persist safe IDs through `updateOpenAIMetadata`; do not return providerTrace or raw vendor bodies in route response. |
| T-10-03-04 | Prompt Overreach | edit prompt | mitigate | Route delegates prompt wrapping to Plan 02 `generatePartialEdit`, which has strict preserve sections. |
| T-10-03-05 | Denial of Service | synchronous OpenAI edit route | mitigate | Extend route timeout to 120 seconds before the OpenAI Image API call and surface key/config errors clearly. |
| T-10-03-06 | Provider Confusion | forged edit payload | mitigate | Ignore request-body provider fields and branch only from persisted `generation.provider`; tests cover forged payload attempts. |
</threat_model>
